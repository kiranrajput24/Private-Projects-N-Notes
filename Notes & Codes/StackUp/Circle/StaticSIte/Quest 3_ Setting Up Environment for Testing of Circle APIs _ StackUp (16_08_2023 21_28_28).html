Quest Walkthrough
Step 1: Sandbox and Production Environments
A sandbox is an isolated testing environment that allows developers to run and test their programs without affecting the main application, system or platform. The sandbox environment is perfect for prototyping and integration testing as it mimics the production environment. The production environment is where the latest version of software, products and updates are deployed. This is the environment where the end users can experience and interact with the product. 
The Circle APIs are available on the sandbox and production environment and are connected to the testnet and mainnet respectively. The mainnet is the main network where all the changes are deployed after being tested. The testnet is a test network where developers can test out new smart contracts and applications. Using these sandbox APIs, you can write code to interact with it, do testing and seamlessly switch to the production APIs when you are ready to go live with your code.

Another advantage of the sandbox environment is that they do not charge payment nor do they use any real USDC. Instead, they use USDC on the test network and thus have no value and cannot be redeemed for USD.
Step 2: Create Sandbox Account
Now that you understand what a sandbox environment is and its uses, we will go through how you can create a sandbox account. Firstly, you will need to access the signup page. You will need to enter the required information on the registration page. 
Please note that you will also need to set up the 2FA (2-Factor Authentication) which will require you to download an application from the Apple App Store or the Google Play Store. Instructions will be provided on the page on how you can set up the 2FA. 2FA verifies your identity using an extra form of verification other than your login credentials. In this case, it is the 6-digit number from the authenticator application.

Once you have successfully signed up for your account, you will be shown the home page, similar to the 'expected output', except that you should have no USDC and no recent activity. 

Congratulations! With this, you have successfully set up your sandbox account.

Step 3: Generate an API Key
In this step, we will go through how you can generate an API key for your Circle Sandbox account. The API key is used to authenticate API requests and check for an authorized user. Continuing from the previous step, you will need to click the “Developer” button on the left-hand-side and “Get a new API Key”.
Clicking on that button will generate a new API key. Please take down your API key and keep it safe. 

To complete this quest, please take a screenshot of the entire page with your newly generated API key with part of the key obscured for security reasons. 

Your screenshot should show:
